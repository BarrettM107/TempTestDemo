### 7. 控制台错误信息检查

**状态：** 发现警告（非致命错误）

#### 发现的问题：

**问题 1：大量 "读取无效地址" 警告**

- **描述：** 当 CPU 执行指令时，MMU 会输出大量 `[MMU] 读取无效地址: 0x...` 的警告信息
- **原因：** 模拟 ROM 数据量过小（仅 192 字节），CPU 执行时 PC 超出 ROM 有效范围
- **影响：** 仅为警告信息，不会导致程序崩溃
- **建议：** 使用真实 GBA ROM 文件进行测试，或增加模拟 ROM 数据量

**问题 2：缺少 BIOS 文件**

- **描述：** 项目未包含 GBA BIOS 文件
- **影响：** 部分需要 BIOS 的功能（如 SWI 中断处理）可能无法正常工作
- **建议：** 如需完整模拟，需额外加载 GBA BIOS 文件

## 五、修复建议

### 建议 1：优化无效地址警告输出

**文件：** `js/emulator.js`  
**位置：** MMU 类的 `resolveAddress` 和 `read*` 方法

**问题：** 当 ROM 未加载或数据不足时，控制台会输出大量警告信息，影响调试体验。

**修复方案：**
```javascript
// 在 MMU 类中添加一个标志控制警告输出
constructor() {
    // ... 现有代码 ...
    this.verbose = false; // 添加 verbose 标志
}

// 在读取无效地址时，根据 verbose 标志决定是否输出警告
read8(address) {
    // ... 现有代码 ...
    default:
        if (this.verbose) {
            console.warn(`[MMU] 读取无效地址: 0x${address.toString(16).padStart(8, '0')}`);
        }
        return 0;
}
```

### 建议 2：添加 BIOS 加载提示

**文件：** `index.html` 或 `js/emulator.js`

**修复方案：** 在 UI 上添加 BIOS 文件上传选项，或在 ROM 加载时检查 BIOS 是否存在并给出友好提示。

### 建议 3：完善 PPU 渲染模式

**文件：** `js/emulator.js`

**问题：** 当前模式 0/1/2 使用测试图案，模式 3/4/5 虽有实现但较为简化。

**修复方案：** 逐步实现完整的平铺背景渲染（Mode 0-2）和精灵渲染系统。

### 建议 4：添加 ROM 文件大小验证
